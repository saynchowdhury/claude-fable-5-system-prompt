# Contributing to Claude Fable 5 System Prompt

Thank you for your interest in contributing! This repository is a living archive and research hub for understanding Anthropic's Mythos-class models. We welcome contributions from researchers, developers, and AI enthusiasts.

<!-- Curated by Sayan Chowdhury -->

## How You Can Contribute

Because the raw system prompt is a static artifact (until Anthropic updates it), contributions primarily focus on **analysis, tooling, and documentation**.

### 1. Submit New Analysis or Findings
Have you found an interesting detail in the system prompt? Discovered a novel jailbreak or behavioral quirk? 
- Open an Issue using the **Analysis Submission** template.
- If you have written a detailed markdown report, submit a Pull Request adding it to the `system-prompt/analysis/` or `highlights/` directories.

### 2. Prompt Updates
If you notice that Claude Fable 5's behavior has changed significantly or you have extracted a newer version of the system prompt:
- Open an Issue using the **Prompt Update** template.
- Provide the new prompt text and the method used to extract it so we can verify and update the repository.

### 3. Interactive Tools and Visualizations
We welcome PRs that add interactive components to the repository, such as:
- Scripts to parse or visualize the prompt JSON tools.
- Comparison charts against other models (e.g., GPT-5.5, Gemini 3).
- UI wrappers for exploring the prompt (e.g., a GitHub Pages site).

## Contribution Guidelines

1. **Maintain Formatting:** Please ensure your Markdown files are well-formatted and use standard GitHub Flavored Markdown.
2. **Attribution:** When adding new files, please include appropriate attribution for your research, while maintaining the repository's baseline curation footers.
3. **No Malicious Content:** While we discuss safety classifiers and dual-use capabilities (Fable vs. Mythos), do not submit PRs containing weaponized exploits or dangerous code generated by the model. Keep discussions academic and focused on the model's architecture.

## Getting Started

1. Fork the repository.
2. Create a new branch (`git checkout -b feature/your-analysis`).
3. Commit your changes (`git commit -m 'Add analysis on tool use'`).
4. Push to the branch (`git push origin feature/your-analysis`).
5. Open a Pull Request!

We review PRs regularly and appreciate your help in making this the premier resource for Claude Fable 5 research.
